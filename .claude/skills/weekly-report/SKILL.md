# 주간 작업 보고서 생성

## 실행 절차

### Phase 1: 기간 확인
1. 오늘 날짜 기준으로 **지난 주** 월~일 기간 산정 (ISO 8601 주차)
2. 사용자에게 기간 확인: "2026.07.07 ~ 07.13 (W28) 보고서를 생성합니다. 맞으신가요?"
3. 사용자 이름 확인 (메모리에 없으면 질문)

### Phase 2: Git 활동 수집
1. 사용자의 홈 디렉토리에서 모든 Git 레포 탐색
2. 각 레포에서 해당 기간의 커밋 수집:
   ```bash
   git log --oneline --since="{시작일}" --until="{종료일}"
   ```
3. 커밋 메시지에서 작업 유형 자동 분류:
   - `feat:` → 신규 기능
   - `fix:` → 버그 수정
   - `refactor:` → 리팩토링
   - `docs:` → 문서
   - `ci:` → CI/CD
   - 한국어 커밋도 내용 기반 분류
4. 프로젝트별 커밋 수, 주요 변경 사항 정리

### Phase 3: 비코드 작업 확인
1. 사용자에게 질문:
   - "이번 주 코드 외 작업이 있었나요? (분석, 미팅, 기획, 하네스 운영 등)"
   - "이슈나 블로커가 있었나요?"
   - "다음 주 계획은?"
2. 답변을 보고서에 반영

### Phase 4: 보고서 생성
1. `RULES.md`의 템플릿에 맞춰 보고서 작성
2. 파일 저장: `reports/{연도}/W{주차}/{이름}.md`
3. 사용자에게 내용 확인 요청
4. 확인 후 커밋 & 푸시 (사용자 동의 시)

## 보고서 품질 기준

- 각 프로젝트의 작업 내용은 **구체적**으로 (커밋 메시지 기반 + 맥락 보강)
- 성과는 **정량적**으로 (커밋 수, 기능 수, 배포 여부)
- 상태는 반드시 완료/진행중/보류 중 택 1
- 총 분량: A4 1~2페이지 (너무 길지 않게)

## 취합본 생성 (관리자 전용)

`취합본 생성` 또는 `SUMMARY 만들어줘` 요청 시:
1. 해당 주차 폴더의 모든 `*.md` 파일 읽기 (SUMMARY.md 제외)
2. `RULES.md`의 SUMMARY.md 구조에 맞춰 취합본 생성
3. 팀원별 하이라이트 1줄 요약, 공통 이슈, 다음 주 포커스 정리
4. `SUMMARY.md`로 저장
