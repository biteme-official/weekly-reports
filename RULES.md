# 작업 보고서 규칙

## 개요

주간/월간 작업 보고서를 Claude Code로 자동 생성하여 이 레포에 업로드한다.
관리자는 GitHub Pages 뷰어에서 전체 팀 현황을 확인하고, 필요 시 취합본을 생성한다.

## 폴더 구조

```
weekly-reports/
  RULES.md                 ← 이 파일 (작성 규칙)
  CLAUDE.md                ← Claude Code 프로젝트 설정
  reports/
    2026/
      W29/                 ← 주간 보고서 (ISO 주차, 월요일 기준)
        홍길동.md
        김철수.md
        SUMMARY.md         ← 관리자용 취합본
      M07/                 ← 월간 보고서 (7월)
        홍길동.md
        SUMMARY.md
```

## 파일 작성 규칙

### 파일명
- 주간: `reports/{연도}/W{주차}/{이름}.md` — 예: `reports/2026/W29/홍길동.md`
- 월간: `reports/{연도}/M{월}/{이름}.md` — 예: `reports/2026/M07/홍길동.md`

### 기간 기준
- **주간:** ISO 8601 주차 (월요일~일요일). 매주 월요일에 지난 주 보고서 작성
- **월간:** 매월 1일에 지난 달 보고서 작성 (또는 수시 요청 시 생성)

### 작성 방법
1. 이 레포를 clone 또는 pull
2. 자신의 프로젝트 디렉토리에서 Claude Code 실행
3. `/weekly-report` 입력 (또는 아래 템플릿 직접 작성)
4. 생성된 MD 파일을 이 레포의 해당 주차 폴더에 복사
5. commit & push

## 데이터 수집 범위

보고서 생성 시 아래 소스를 모두 확인한다:

### 자동 수집 (Git 기반)
- `git log --since` 으로 해당 주간 커밋 이력
- 커밋 메시지에서 작업 유형 분류 (feat/fix/refactor/docs/ci)
- 프로젝트별 커밋 수, 변경 파일 수

### 수동 보완 필요
- 비코드 작업: 데이터 분석, 기획, 미팅, 의사결정
- Claude 하네스 운영: CM 리뷰, KPI 사이클, 파트너 인텔리전스 등
- 외부 시스템 작업: Slack, Notion, Google Sheets, Amplitude 등
- 이슈/블로커 및 다음 주 계획

## 관리자용: 취합본 생성

관리자는 해당 주차 폴더의 모든 개인 보고서를 기반으로 `SUMMARY.md`를 생성할 수 있다.

### SUMMARY.md 구조
```markdown
# W{주차} 주간 취합 보고서

- **기간:** {시작일} ~ {종료일}
- **작성일:** {날짜}
- **참여:** {인원 수}명

## 전체 요약
> 팀 전체 주요 성과 3줄 요약

## 팀원별 하이라이트
| 팀원 | 주요 작업 | 상태 |
|------|----------|------|
| 홍길동 | {핵심 작업 1줄} | 정상 |
| 김철수 | {핵심 작업 1줄} | 블로커 있음 |

## 공통 이슈
- {팀 차원 이슈}

## 다음 주 팀 포커스
- {우선순위 1}
- {우선순위 2}
```

## 작성 가이드라인

1. **구체적으로** — "작업함" 대신 "결제 페이지 Storefront API 토큰 캐싱 적용, 체크아웃 성공률 측정 시작"
2. **정량적으로** — 커밋 수, 완료율, 배포 횟수, 에러 감소율 등 숫자 포함
3. **상태 명확히** — 완료/진행중/보류를 반드시 표기
4. **블로커는 즉시** — 다음 주까지 기다리지 말고 발견 즉시 이슈에 기록
5. **회고는 솔직하게** — 잘한 점과 개선점 균형 있게 작성
